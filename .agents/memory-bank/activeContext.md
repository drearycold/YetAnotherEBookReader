## Recent Changes & Decisions
- **P1/A09 DSReaderHelperConnector Thread Safety Fixed (2026-06-18):** Removed the `DispatchQueue.main.sync` call and the redundant `URLCredentialStorage.shared.set(...)` block from `DSReaderHelperConnector.urlSession`. The `CalibreServerTaskDelegate` already handles HTTP authentication via its `urlSession(_:task:didReceive:completionHandler:)` delegate callback, so the manual credential storage was both unnecessary and a potential indirect deadlock source when accessed from background threads (e.g., `ReadingSessionManager.updateCurrentPosition`). The `urlSession` computed property now simply delegates to `calibreServerService.urlSession(server:)`, which returns a cached, delegate-backed session. Also eliminates the per-access overhead of re-setting credentials. Added `DSReaderHelperConnectorTests` (3 cases) verifying that `urlSession` returns a valid session and can be accessed from multiple background threads without blocking.
