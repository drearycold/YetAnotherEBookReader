- **Shelf Ad Placement And Pure Layout Planning (2026-07-10, complete):**
  Restored the Google Mobile Ads shelf integration with viewport-aware
  placement: Recent uses regular-width native end-caps when the row remains
  content-led, while compact or unavailable-native layouts use inline adaptive
  banners; Discover places native strips between sections with the same compact

  and Discover reserve bottom space with real `safeAreaInset` exclusion.
  `ShelfLayoutPlanner` then extracted column calculation, row packing, filler,
  ad insertion, and content-height planning into pure Foundation/CoreGraphics
  value types. Recent and Discover render deterministic plans without changing
  existing book interaction behavior. Discover element IDs are namespaced as
  `section:`, `ad:`, and `filler:` to prevent collisions with server-controlled
  section IDs. Focused shelf/ad tests pass (28 tests), `git diff --check`
  passes, and the iOS Simulator build succeeds. Committed as `d50042f1`
  (`refactor: extract shelf layout planner`).

- **Comprehensive Test Failures Fix (2026-06-26, complete):** Four follow-up commits resolve a cluster of test failures reported after the test-realm refactor landed. Three independent root causes were at play: (1) `AppContainer.swift:222` force-unwrapped `libraryManager.calibreLibraries.first!.value`, which crashed with 'Unexpectedly found nil' whenever the bootstrap returned an empty library list. The trigger was the iOS Simulator's per-process file-descriptor limit: each `MockAppContainerFactory.makeContainer` call opened a new in-memory main Realm, and ~339 tests exhausted the limit (last test to hit the cap was `UnifiedCategoryServiceTests.testFetchAndCacheCategorySuccess` with 'Too many open files'). Replaced the force-unwrap with a `guard let` that skips the mock-data population block on bootstrap failure; the container still constructs and any test not depending on the populated mock data continues to pass. (2) Added a per-testName cache to `MockAppContainerFactory` so the in-memory main Realm and `InMemoryServerScopedRealmConfigurationProvider` are reused across tests in the same class. `AppContainer` instances remain fresh per test (so per-instance in-memory state is reset), but file descriptors are no longer per-test. Updated ~20 test files to use class-scoped `testName` values (`"ClassName"` instead of `"ClassName-\(UUID().uuidString)"`); `BookDetailViewModelTests.swift` had four inner test classes (ReadingPositionViewModelTests, ActivityListViewModelTests, LibraryViewModelTests, ReadingPositionRepositoryThreadingTests) all incorrectly using the `BookDetailViewModelTests-` prefix, renamed each. `MockAppContainerFactoryTests` is unchanged because its tests already use distinct testNames and explicitly assert per-call isolation; the new cache is a no-op for them. (3) `SectionShelfViewModel` used `.collect(.byTime(RunLoop.main, .seconds(1))).receive(on: DispatchQueue.main)` for batching. Both operators require their target scheduler to be running on the main thread, but `async` tests on the MainActor yield the main thread to async machinery, not the RunLoop or main dispatch queue. Removed both operators; the sink is now wired directly to `discoverShelfItemsSubject` and fires synchronously on the send thread. All current `send` callers are on the main thread (YabrShelfDataModel.notifyDiscoverShelfChanged is `@MainActor`; addToShelf wraps its send in `Task { @MainActor in }`), so the synchronous sink runs on the main thread in production. The 1-second batching buffer was a perf optimization that just deferred applyFiltering; removing it makes the viewModel more responsive. Tests `SectionShelfViewModelTests.testUpdateShelfModels` and `ShelfDisplayModelsTests.testSectionShelfViewModelMappingAndFilters` changed from `async`/`await fulfillment`/`Task.sleep` to synchronous assertions immediately after `send` — the sink is now synchronous, and the assertion must happen before the YabrShelfDataModel bootstrap's later "Author: Unknown" emission overwrites displaySections. (4) `UnifiedSearchIntegrationTests.testUnifiedSearchEndToEndWithMockServer` sink was fulfilling the expectation on every non-empty emission, which the publisher can do more than once (cache hit then network result, or two cache hits). Added `&& finalResult == nil` guard so the expectation is fulfilled only once. Full unit suite: 339/339 green; iOS Simulator and Mac Catalyst builds succeed.
