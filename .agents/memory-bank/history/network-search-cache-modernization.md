- **Intermittent Swift Actor Runtime Trap Recorded (2026-07-01, unresolved):** During app startup, Xcode stopped once with `EXC_BREAKPOINT` at `(anonymous namespace)::DefaultActorImpl::unlock` / `swift_task_switchImpl`. Repeated attempts did not reproduce it, and the captured Task 1320 stack contained no application frame or console fatal-error text, so no root cause is confirmed. Concurrent neighboring tasks were mapping Realm-backed category/search cache values in `RealmSearchCacheStore`, but each visible path opened and consumed a queue-local Realm before suspension; their presence is correlation, not evidence that Realm caused the trap. Two concurrency risks remain candidates if the issue recurs: `RealmSearchCacheStore` is `@unchecked Sendable` while resolving mutable `container.realmConf` and `container.calibreLibraries` from cooperative tasks, and background shelf work may read manager dictionaries such as `booksInShelf` while the main thread publishes mutations. On recurrence, capture the first console runtime diagnostic, whether Continue resumes or terminates the process, the complete crashing-task backtrace, and a reproduction under Thread Sanitizer or Address Sanitizer before changing code.
