# ModelData Elimination History

Archived records for ModelData Elimination Phase 4 and earlier phases, AppContainer introduction, protocol migration, and final ModelData deletion.

Source: split from `.agents/memory-bank/activeContext.md` on 2026-07-03. Entries are preserved as historical records unless this file explicitly says otherwise.

## Recent Changes & Decisions

- **Phase 4d: ModelData + CalibreServerConfigProvider Deleted (2026-06-25):** Deleted `Models/ModelData.swift` (382 lines) and `Models/CalibreServerConfigProvider.swift` (24 lines). `AppContainer` is now the sole composition root. `AppContainerProtocol` absorbed the deleted protocol's members (`updateBook`, `getPreferredFormat`, plus the runtime state already added in 4a+4b). `CalibreServerService.config` and `init` now take `AppContainerProtocol` instead of `CalibreServerConfigProvider`. Xcode project updated to drop the two deleted file references from iOS app and Mac Catalyst targets. All 329 unit tests pass, iOS Simulator build succeeds. The full Phase 4 ModelData elimination is complete: ModelData's 1,073-line god object is replaced by a 366-line AppContainer + 130-line AppContainerProtocol.
- **Phase 4c: View / ViewModel / Adapter Renamed to AppContainer (2026-06-25):** Bulk-renamed `ModelData` → `AppContainer` and `modelData` → `container` across 36 app source files and 6 test files (95 files, 1,171 insertions / 1,150 deletions). `YetAnotherEBookReaderApp.swift` now creates `AppContainer()` as the sole `@StateObject`. All 16 View files declare `@EnvironmentObject var container: AppContainer`. All 9 ViewModels + the `YabrEBookReaderNavigationController`, `EpubFolioReaderContainer`, and `FolioReaderViewController` adapters take `container: AppContainer`. The 11 manager / repository / init-helper files updated in 4a+4b had their `init` and `setup` parameter label renamed from `modelData:` to `container:` so the composition root call sites still bind. `AuthPlugin` in `Network/ImageRequest.swift`, the `ModelData.shared` access sites in `CalibreCoreModels.swift` / `Views/FolioReaderView/Providers.swift` / `YabrEBookReaderMetaSource.swift` / `YabrReaderNavigationViewModel.swift` / `FolioReaderViewController.swift` / `ReadingPositionRepository.swift`, and the test `setUp` methods all switched to `AppContainer`. The `FolioReaderProviderBookIdTests` helper disambiguates the class-level `container: AppContainer` field from the `EpubFolioReaderContainer` instance by using `epubContainer`. All 329 unit tests pass.
- **Phase 4a+4b: AppContainer Introduced Alongside ModelData (2026-06-25):** Created `Models/AppContainer.swift` (~367 lines) as a verbatim mirror of `ModelData` so both classes can coexist during the remaining Phase 4 sub-steps. Decoupled 11 manager/repository/init-helper files from the concrete `ModelData` class by widening their `init(modelData:)` parameter and `weak var modelData` property types to `AppContainerProtocol`. Extended `AppContainerProtocol` to add runtime state (`calibreLibraries`, `calibreServers`, `calibreServerInfoStaging`, `librarySyncStatus`, `booksInShelf`, `deviceName`), sync progress (`updatingMetadata*`), the missing database-lifecycle and activity-log methods (`getBook`, `refreshDatabase`, `tryInitializeDatabase`, `initializeDatabase`, `migrateLegacyReadPosData`, `getCustomDictViewer`, `getCustomDictViewerNew`, `updateCustomDictViewer`, `cleanCalibreActivities`, `logStartCalibreActivity`, `logFinishCalibreActivity`), and `LibraryResolver`/`ServerResolver` conformances. Converted the three `ModelData` extensions in `ShelfDataManager.swift` (`registerRecentShelfUpdater`, `buildShelfBookItem`, `parseShelfSectionId`) to extensions on `AppContainerProtocol where Self: ObservableObject` so both classes receive them through protocol conformance. All 329 unit tests pass, iOS Simulator build succeeds. The repository-layer and helper-layer are now container-agnostic; Phase 4c (View injection) can follow.
- **ModelData Elimination Phase 3 Completed (2026-06-25):** Introduced `Models/AppContainerProtocol.swift` (~95 lines) — a single facade protocol aggregating all repositories, managers, services, subjects, and runtime state. `ModelData` now conforms to `AppContainerProtocol` alongside the existing `CalibreServerConfigProvider` and `LibraryProvider`.
