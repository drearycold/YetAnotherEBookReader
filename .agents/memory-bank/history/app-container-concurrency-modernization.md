## Recent Changes & Decisions

- **Review Follow-up: Settings Server Status Refresh and Async Broadcaster Registration (2026-07-03, complete):** Fixed two review findings from the AppContainer/manager async migration. `ManagerAsyncBroadcaster` now registers continuations before yielding the initial value, closing a small subscription race. `SettingsViewModel` now consumes `CalibreServerManager.serverInfoStagingSnapshots()` into published view-model state, so Settings server refresh/probe status and reachability row text update even though `AppContainer` is no longer an `ObservableObject`. Added focused regression coverage for Settings server-info snapshots and broadcaster initial/subsequent delivery. Validation: focused `SettingsViewModelTests` and `CalibreServerManagerTests` passed (21 tests); `git diff --check` passed; standard iOS Simulator build passed.
- **Managers Combine Boundary Removal (2026-07-02, complete):** Removed Combine from `Models/Managers/` as a layer dependency. `CalibreBookManager`, `CalibreLibraryManager`, `CalibreServerManager`, `ReadingSessionManager`, and `FontsManager` are now plain classes with explicit `AsyncStream` snapshot/state outputs backed by a small app-local `ManagerAsyncBroadcaster`; manager event buses now use direct methods plus async streams. AppContainer legacy subjects and later `ObservableObject` / `@Published` compatibility were removed in follow-up cleanups. Migrated affected Settings/Server/Library view models and manager tests from manager `$property`/subject subscriptions to cancellable stream-consuming tasks. Validation: Managers Combine scan passed with no matches; `git diff --check` passed; focused manager/view-model tests passed (63 tests); standard iOS Simulator build passed.
- **ModelData Main Thread Notification Forwarding:** Integrated `.receive(on: DispatchQueue.main)` on manager `objectWillChange` subscriptions in `ModelData.init` to ensure all SwiftUI state-propagation updates originating from nested managers are delivered on the main thread, eliminating runtime background thread update warnings.
- **Cleanup of Duplicates:** Removed duplicate physical folders (`Managers 2`, `Managers 3`, `Managers 4`) from the workspace.
- **Context Guardrails:** Established the `.agents/memory-bank` directory to provide strict architectural guidelines, preventing subagents from hallucinating or deviating from the Swift Package Manager / SwiftUI architecture.
