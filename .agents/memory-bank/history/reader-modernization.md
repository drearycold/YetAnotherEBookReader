# Reader Modernization History

Archived records for Readium, FolioReader, PDF reader, reading position, highlights, preferences, and Book Detail reader work.

Source: split from `.agents/memory-bank/activeContext.md` on 2026-07-03. Entries are preserved as historical records unless this file explicitly says otherwise.

## Recent Changes & Decisions

- **Cross-Engine Reader Preference Compatibility (2026-07-07, complete):** Standardized `ReaderEnginePreferences.themeMode` as shared canonical values (`0 light`, `1 sepia`, `2 green`, `3 dark`, `4 night`) and moved engine raw theme mapping to the Readium, FolioReader, and YabrPDF adapter boundaries. `ReaderPreferenceRepository.loadInitialPreferences` now prefers target-engine preferences, then falls back through compatible saved preferences from other engines by target reader type. Folio dark/night saving now stores the shared raw theme and derives `nightMode` from shared dark-or-later while preserving legacy fallback for incomplete rows. Follow-up restored Folio native preference completeness by adding full per-book `FolioReaderPreferenceValue` persistence for all FolioReaderKit-backed preference keys and keeping `ReaderEnginePreferences` as a lossy compatibility subset only. Focused validation passed: `ReaderPreferenceRepositoryTests`, `ReadiumPreferenceValueTests`, `YabrPDFViewControllerTests`, and `FolioReaderProviderBookIdTests` (65 tests).
- **Views Combine Subscription Removal (2026-07-02, complete):** Removed explicit Combine subscriptions/imports from `Views/` while preserving `@Published` / `ObservableObject` as SwiftUI state. `AppContainer` exposes async event bridges for dismiss-all, book-import, and reader scene-phase activity; the legacy subject forwarding that still existed at this stage was removed in the later AppContainer legacy subject cleanup. Migrated `LibraryInfoView`, `YabrEBookReaderNavigationController`, `MainViewModel`, `ReaderOptionsViewModel`, `SettingsViewModel`, and `ServerViewModel` from `sink`/`AnyCancellable` to `Task` / async streams or `didSet` state handling, and updated event senders to use publish methods. Validation: `Views/` Combine scan passed with no matches for imports/subscriptions; `git diff --check` passed; focused `MainViewModelTests`, `ReaderOptionsViewModelTests`, `SettingsViewModelTests`, and `BookDetailViewModelTests` passed (41 tests; no standalone `ServerViewModelTests` suite exists); standard iOS Simulator build passed.
- **ActivityListViewModel Stream Test Race Fix (2026-07-02, complete):** Fixed the failing `ActivityListViewModelTests.testInitializationUsesRepositoryAndReceivesUpdates` after the async repository migration. `ActivityListViewModel` now creates the repository activity stream before launching its consuming task and updates `@Published` activities from a main-actor task. The mock repository exposes observer subscriber count, and the test now waits on observer registration/state conditions instead of a fixed `RunLoop` delay. Validation: focused `ActivityListViewModelTests.testInitializationUsesRepositoryAndReceivesUpdates` plus `BookDetailViewModelTests` passed (25 tests); `git diff --check` passed; standard iOS Simulator build passed.
- **Stage P12 (Performance Plan): Cache Server-Scoped Configurations (2026-06-28, complete):** Refactored `DefaultServerScopedRealmConfigurationProvider` to cache `Realm.Configuration` instances using a thread-safe cache keyed by `serverUUID` and `schemaVersion`. Implemented double-checked locking using `NSLock` to perform expensive configuration setup outside the lock when missing. Added focused tests in `DefaultServerScopedRealmConfigurationProviderTests.swift` verifying caching, server differentiation, schema modification behavior, and multi-thread concurrent safety. Checked cache status for all book formats in-memory, avoided multiple calls to `updateBook` in loops (now at most one save per book), and eliminated startup debugging print statements. All 378 unit tests and 1 UI test passed.

- **FolioReader Default Profile Behavior Restored (2026-06-23):** Reconnected the database-backed profile store to `FolioReaderDelegatePreferenceProvider` (Phases 1-3). To resolve this, we extended `FolioReaderDelegatePreferenceProvider` to accept `profileRealmConfig` and pass `modelData?.realmConf`. - **FolioReader Position Restore Race Fix Implemented (2026-06-23):** Fixed an intermittent FolioReader reading-position restore failure caused by `RealmReadingPositionRepository.savePosition(_:forBookId:)` deleting old same-identity positions in one Realm write transaction and adding the replacement in a second transaction. FolioReaderKit saves `savedPositionForCurrentBook` asynchronously in `viewWillAppear`, then immediately calls `reloadData()` and synchronously asks the provider for the current position; the previous two-transaction save path could expose a momentary zero-row state to `getPosition`. `savePosition` now performs stale-position deletion, same-identity existence check, and replacement insert inside one Realm write transaction. Added regression coverage in `FolioReaderProviderBookIdTests` for atomic older-position replacement, provider restore after replacement, and concurrent save/read behavior that must not return nil. Validation: focused `FolioReaderProviderBookIdTests` passed, and full `xcodebuild test -project YetAnotherEBookReader.xcodeproj -scheme YetAnotherEBookReader -sdk iphonesimulator -destination 'platform=iOS Simulator,name=iPhone 17' -derivedDataPath /tmp/YabrDerivedData` passed.
- **FolioReader Highlight Position Restore Fix Implemented (2026-06-23):** Implemented the fix from `.agents/plans/FOLIO_READER_HIGHLIGHT_POSITION_RESTORE_IMPLEMENTATION.md`. The `BookHighlight.toFolioReaderHighlight()` extension in `Views/FolioReaderView/Providers.swift` now calls `highlight.encodeContents()` before returning, so `contentEncoded`, `contentPreEncoded`, and `contentPostEncoded` are populated. FolioReaderKit's `Bridge.js` `injectHighlight()` calls `decodeURIComponent(oHighlight.contentEncoded)` on every injected highlight; without encoded content the decode throws, aborts the highlight batch, and blocks the page load that also restores reading position. Also fixed `EpubFolioReaderContainer.applyHighlights(_:)` to obtain the provider through `folioReaderHighlightProvider(self.folioReader)` instead of only consuming the optional cached property, so persisted highlights are never dropped when `applyHighlights` runs before the provider exists. Added a fileprivate `ensureEncoded` last-line guard applied to every `FolioReaderHighlight` returned to FolioReaderKit (`getById`, `allByBookId andPage`, `all`). Added 4 regression tests in `FolioReaderProviderBookIdTests`: `testBookHighlightToFolioReaderHighlightPopulatesEncodedContent`, `testHighlightProviderReturnsEncodedHighlightsForFolioReaderRuntimeId`, `testEpubFolioReaderContainerApplyHighlightsCreatesProviderWhenNil`, and `testCombinedPositionAndHighlightRestoreViaFolioReaderRuntimeId`. Validation: focused `FolioReaderProviderBookIdTests` 11/11 green, full `xcodebuild test` green (191 tests).
